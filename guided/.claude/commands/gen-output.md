# /project:gen-output

根據規格書與範本，生成對應的文件到 `output/` 資料夾。

## 執行步驟

1. 讀取 `specs/my-spec.md`
   - 若檔案不存在，告知使用者先說「我想討論規格」

2. 確認資源
   - 列出 `templates/` 中的可用範本
   - 列出 `assets/` 中的可用素材
   - 若有缺少的素材，告知使用者並繼續（留佔位符）

3. 根據規格書的「產出類型」選擇工具：
   - `.pptx` → 使用 `pptxgenjs`（Node.js 腳本）
   - `.docx` → 使用 `python-docx`（Python 腳本）
   - `.md` / `.txt` → 直接寫檔

4. 生成前先向使用者確認：
   > 「我將根據你的規格生成以下內容：[列出清單]。確認開始嗎？」

5. 生成完畢後：
   - 告知輸出路徑
   - 簡述生成了哪些檔案
   - 詢問是否需要調整

## 輸出路徑規則

- 單一文件：`output/檔名.副檔名`
- 多個文件：`output/專案名稱/`

## 風格規則

- 若 `templates/` 有範本，解析範本的顏色、字型、版面後套用
- 若沒有範本，使用乾淨的預設設計（白底、深色文字、適當的標題色）
- 素材圖片若不存在，使用帶文字說明的佔位色塊（不要留空白）
