# Claude 簡報生成框架

用 Claude Code 透過對話生成 `.pptx` 簡報。不需要工程背景，開口就能做。

---

## 先選一個模式

| | quick／快速版 | guided／規格書版 |
|--|-------------|----------------|
| **適合** | 一次性、臨時性簡報 | 需要重複製作的系列簡報 |
| **例如** | 業務報告、會議說明、活動簡介 | 課程簡報、品牌提案、固定格式月報 |
| **前置設定** | 不需要 | 需要先建立規格書（一次） |
| **每次使用** | 告訴 Claude 主題和內容 | 只需告訴 Claude 這次的內容 |
| **資料夾** | `quick/` | `guided/` |

不確定選哪個？**先用 quick**，之後有需要再搬到 guided。

---

## Quick 快速版

### 適用情境
臨時被主管要一份業務報告、活動說明要趕出來、一次性的客戶簡報。

### 使用流程

```
1. 用 Claude Code 打開 quick/ 資料夾
2. 跟 Claude 說：「我想做一份簡報」
3. Claude 會問你目的和內容
4. Claude 規劃好每頁大綱後，讓你確認
5. 確認後開始生成，完成後可以直接預覽
```

### 你可以說的話

| 說這個 | 會發生這件事 |
|-------|------------|
| 「我想做一份簡報」 | 開始討論內容，確認大綱後生成 |
| 「幫我檢查環境」 | 確認 Node.js 和套件是否安裝好 |
| 「預覽簡報」 | 在 Claude 對話視窗裡直接看成品 |

### 資料夾結構
```
quick/
├── CLAUDE.md          ← Claude 的工作說明（不用動）
├── specs/
│   └── style-guide.md ← 視覺設定（顏色、字型，可自訂）
├── assets/            ← 把你的 logo 和圖片放這裡
├── templates/         ← 把參考範本放這裡（可不放）
└── output/            ← 生成的簡報會出現在這裡
```

## Guided 規格書版

### 適用情境
你要做 30 堂課的課程簡報，每堂格式一樣、對象一樣、用語規範一樣，只有內容每次不同。
或是你有固定的品牌風格、公司用語規範，不想每次都重新說明。

### 核心概念

**規格書 = 每次都一樣的東西**，例如：
- 目標對象（這份簡報是給誰看的）
- 固定結構（每頁都有學習目標 → 內容 → 小測驗）
- 用語規範（不說「同學」，說「學習者」）
- 品牌色、logo、字型

**每次輸入 = 這次特定的內容**，例如：
- 這堂課的主題
- 這次要講的重點
- 這份報告的數據

建立好規格書後，往後每次只需告訴 Claude「這次的內容是什麼」，其他都自動套用。

### 使用流程

**第一次使用（建立規格書）：**
```
1. 用 Claude Code 打開 guided/ 資料夾
2. 說：「幫我檢查環境」
3. 說：「我想討論規格」
4. Claude 會引導你想清楚哪些東西要放進規格書
5. 討論完後，規格書自動存到 specs/my-spec.md
```

**之後每次使用：**
```
1. 打開 guided/ 資料夾
2. 說：「幫我生成簡報」
3. 告訴 Claude 這次的主題和內容
4. 確認大綱後生成，完成後預覽
```

### 你可以說的話

| 說這個 | 會發生這件事 |
|-------|------------|
| 「介紹這個專案」 | 顯示使用說明 |
| 「幫我檢查環境」 | 確認 Node.js 和套件是否安裝好 |
| 「我想討論規格」 | 引導你建立規格書，幫你分清楚什麼該放進去 |
| 「幫我生成簡報」 | 套用規格書，只需補充這次的內容 |
| 「預覽簡報」 | 在 Claude 對話視窗裡直接看成品 |

### 資料夾結構
```
guided/
├── CLAUDE.md                  ← Claude 的工作說明（不用動）
├── specs/
│   ├── style-guide.md         ← 視覺設定（顏色、字型）
│   ├── SPEC-TEMPLATE.md       ← 規格書空白模板
│   ├── my-spec.md             ← 你的專案規格（討論後自動生成）
│   ├── slide-types.md         ← 各頁型說明
│   ├── industry-knowledge.md  ← 產業背景知識（可自行填寫）
│   ├── company-guidelines.md  ← 公司用語與規範（可自行填寫）
│   └── content-principles.md  ← 內容寫作原則（可自行填寫）
├── assets/                    ← logo 和圖片放這裡
├── templates/                 ← 參考範本放這裡
└── output/                    ← 生成的簡報會出現在這裡
```

## 常見問題

**Q：我沒有公司 logo 或品牌色，可以用嗎？**
可以。不放任何素材也能生成，Claude 會用預設的視覺風格。

**Q：生成的簡報可以用 PowerPoint 打開嗎？**
可以，生成的是標準 `.pptx` 格式，PowerPoint 和 LibreOffice 都可以開啟編輯。

**Q：規格書可以修改嗎？**
可以。直接打開 `specs/my-spec.md` 編輯，或告訴 Claude「幫我修改規格書」。

**Q：不確定該用 quick 還是 guided？**
先用 quick，等你發現自己在做重複性的簡報時，再考慮建立 guided 的規格書。
