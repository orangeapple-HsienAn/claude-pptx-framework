# /project:discuss-spec

透過對話引導使用者建立規格書，完成後存到 `specs/my-spec.md`。

## 執行步驟

1. 先問這一題，等對方回答：
   > 「你好！我們來一起建立你的專案規格。首先，你想要生成什麼類型的文件？（例如：簡報、Word 報告、教材、手冊⋯）」

2. 接著問：
   > 「你有沒有以前做過類似的簡報，可以讓我先參考一下？直接上傳到對話裡，或是把檔案放到 `templates/` 資料夾都可以。沒有也沒關係，直接聊也行。」

   - 使用者上傳檔案或放到 `templates/` → 讀取後，簡短說明你看到的內容方向與目的，然後繼續下一題
   - 使用者說沒有 → 直接繼續下一題

3. 根據對方的回答，**依序問以下問題，一次只問一題**：
   - 這份文件是給誰看的？（目標對象）
   - 大概分成幾個主要段落或章節？每個段落的主題是什麼？
   - 有沒有特定的視覺風格？（有範本的話告訴我放在哪，或描述你喜歡的風格）
   - `assets/` 資料夾裡有沒有你要放進文件的圖片或素材？
   - 有沒有特殊格式需求？（頁數限制、語言、特殊元素等）

4. 收集完資訊後，把討論結果整理成規格書並存檔：
   ```
   儲存到：specs/my-spec.md
   ```

5. 存完後說：
   > 「規格書已存到 specs/my-spec.md，你可以隨時打開修改。要現在開始生成文件嗎？跟我說「幫我生成簡報」就會開始。」

## 注意事項

- 問題要有針對性，不要一次全問
- 使用者沒有回答的部分，在規格書裡標記為「待確認」
- 保持對話語氣自然，不要像在填表格
