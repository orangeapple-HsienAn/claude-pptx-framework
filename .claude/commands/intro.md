# /project:intro

歡迎！這個資料夾是一套**用 Claude 生成簡報**的工作框架。
流程是：先跟 Claude 討論你要做什麼、建立規格書，確認方向後再生成簡報。

## 介紹資料夾結構

用對話方式說明（不要貼結構圖）：

- **specs/** — 放視覺設定和規格書。最重要的是 `style-guide.md`（控制顏色和字型），以及你跟 Claude 討論後產出的 `my-spec.md`（你這次要做的簡報規格）。
- **templates/** — 把你的範本放這裡。如果你有現成的 PowerPoint 樣式，丟進來 Claude 就會參考它的設計。
- **assets/** — 圖片、logo 放這裡。放進去之後，生成簡報時會自動嵌入。
- **output/** — 所有生成好的簡報都會出現在這裡。

## 可用功能（直接說就好）

| 你說什麼 | Claude 會做什麼 |
|---------|----------------|
| 「幫我檢查環境」 | 確認 Node.js、pptxgenjs 等套件是否已安裝 |
| 「介紹這個專案」 | 顯示這份說明（你現在看到的） |
| 「我想討論規格」 | 跟 Claude 討論你的需求，建立規格書 |
| 「幫我生成簡報」 | 依規格書生成簡報 |
| 「我想做一份簡報」 | 討論 + 生成一氣呵成（適合已有清晰想法時） |
| 「預覽簡報」 | 開啟生成好的簡報 |

## 標準流程

主動說明建議的使用順序：

1. 說「**幫我檢查環境**」— 先確認環境 OK
2. 說「**我想討論規格**」— 跟 Claude 聊聊你想做什麼，整理成規格書（`specs/my-spec.md`）
3. 看一下規格書，確認方向沒問題
4. 說「**幫我生成簡報**」— 依規格書生成簡報
5. 說「**預覽簡報**」— 預覽成品，有問題再調整

如果你已經很清楚自己要做什麼，也可以直接說「**我想做一份簡報**」，討論完直接生成，省略規格書步驟。

## 確認準備狀況

問使用者這一題，根據回答引導：

> 「在開始之前，有幾件事可以先準備：
> 1. **視覺風格**：`specs/style-guide.md` 預設是紅色 + 橘色的配色。你想沿用這個，還是換成你自己的品牌色？
> 2. **範本（選用）**：如果你有現成的 PowerPoint 範本，可以放到 `templates/` 資料夾。
> 3. **Logo（選用）**：如果有 logo，放到 `assets/` 並命名為 `logo.png`。
>
> 想現在調整風格設定，還是先用預設值試試看？」

**使用者說「先試試看」** → 「好！建議先執行 `/project:check-env` 確認環境，然後輸入 `/project:discuss-spec` 開始跟我討論你的簡報內容。」

**使用者說「要換顏色」** → 讀取 `specs/style-guide.md`，引導修改主色 A、主色 B、字型後說：「設定好了！接著輸入 `/project:discuss-spec` 開始討論簡報內容。」

**使用者問「discuss-spec 和 plan-and-build 差在哪？」** →
「`/project:discuss-spec` 會把討論結果存成規格書，你可以在生成前再確認方向；`/project:plan-and-build` 則是討論完直接生成。第一次用建議走 discuss-spec → 確認規格 → gen-output，比較不容易方向跑掉。」

## 語調原則

- 保持輕鬆，像在幫朋友入門，不是在念說明書
- 一次問一個問題，不要一次丟太多選項
